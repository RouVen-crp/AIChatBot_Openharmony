# AIModelChat 

#### 介绍
目前实现了通过输入同一网段IP+端口连接本地部署模型流程。提供了在Openharmony APP侧（支持平板以及手机）展示UI操作界面与本地模型交互的能力。目前初版已提供高效简洁的界面方便用户与模型交互、历史对话记录保存、新增/删除对话、Ollama本地模型地址手动设置以及配置保存、简单联网搜索、连接状态显示、深色模式、图像识别等功能。

#### 界面交互元素介绍
1.  侧边栏按钮：点击弹出隐藏侧边栏，侧边栏提供对话历史记录，设置等交互按钮。
2.  标题栏：当前对话标题，初次提问内容自动更新。
3.  新增对话按钮：点击新增新对话，可在侧边栏查询新增对话。
4.  状态提示框： 自动查询当前与Ollama部署完本地模型后连接状况。
5.  交互区：用户提问内容输入框，相关增强功能开关按钮，以及提交按钮。

#### 本地模型配置

1.  打开设置相关界面，选择Ollama设置，初次使用时输入url以及端口号，点击连接，会有连接成功与否。
2.  返回上级，选择所需本地模型。
3.  本地模型配置成功后，保证设备与本地模型同一网段。
4.  配置好服务URL地址后选择本地模型若首页状态提示框已连接即可进行对话。

#### 目录说明
   
```markdown
src/
├── main/
│ ├── ets/ 
│ ├── aiCore/  - AI核心功能模块、不同供应商AI客户端统一接口类
│ ├── common/  - 常量
│ ├── component/ - UI组件库
│ ├── db/ - 数据库相关操作
│ ├── entryability/
│ ├── entrybackupability/
│ ├── interfaces/  - 接口定义
│ ├── model/  - 部分通用配置类
│ ├── pages/  - 页面文件
│ ├── preference/  - 用户设置
│ ├── utils/  - 工具类
│ ├── view/ - 视图组件
│ ├── web/  - Web相关功能
│ └── workers/  - TTS/ASR后台工作线程
```
#### 注意事项

1.  思考/结论会以流式输出markdown用于展示，等待输出结果过程中，输出速率与本地模型输出/处理tokens能力以及网络情况有关。
2.  目前仅支持ollama，硅基流动服务商。
3.  本地模型配置成功后，需保证设备与本地模型同一网段。
4.  如有任何BUG或待优化内容，欢迎提交issues。

#### 约束与限制

1.  本示例仅支持标准系统上运行，支持设备：开发者手机二代。
2.  本示例仅支持API12版本SDK。
3.  本示例在IDE版本5.0.0及以上版本已验证通过。
